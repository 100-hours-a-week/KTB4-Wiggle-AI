# HTTP
**HyperText Transfer Protocol**
**정의:** html, css, js, png, jpeg 과 같은 파일이나 문서들이 링크를 통해 서로 연결되어 있는 텍스트 등 일반적인 텍스트를 뛰어넘는 구조화된 텍스트를 전송하기 위해 사용되는 통신 규약
**사용하는 이유:** 웹 서버와 클라이언트가 하이퍼텍스트 문서를 주고받을 수 있도록 정의된 규칙에 따라 문서를 전송하기 위해서
### HTTP message
정의: HTTP를 통해 클라이언트와 서버가 파일 등을 주고 받는 통신의 기본 단위
사용 이유: 클라이언트와 서버가 데이터를 주고 받기 위해서

|순서 | 이름 | 내용                           |
|----|-----|-------------------------------|
| 1  |시작줄(Start Line)| 요청이나 응답의 상태를 나타내는 첫번째 줄|
| 2  |헤더(HTTP Headers)| 메시지 바디를 요약하는 헤더들의 집합    |
| 3  |빈 줄(Empty line) | 헤더와 본문을 구분하기 위해 존재하는 빈 줄입니다.<br>이전까지의 모든 헤더가 전송되었음을 알리고 본문이(body) 부분이 시작된다는 것을 의미합니다.|
| 4  | 본문(Body)       | 데이터나 문서 등 실제 내용이 포함되며 HTML 문서, JSON 데이터 등이 본문에 담길 수 있습니다.|

### HTTP Request Method
**정의**: 클라이언트가 서버에게 어떤 작업을 요청할지 알리는 방법을 정의한 명령 규칙
**사용 이유:** 웹 요청을 보낼 때, URL과 함께 GET, POST, PUT 등의 메소드를 명시하여 서버에게 특정 작업을 요청
- **GET 메서드**
- **POST 메서드**
- **PUT 메서드**
- **PATCH 메서드**
- **DELETE 메서드**
### HTTP Status Code
**정의:** HTTP 요청의 처리 상태를 나타내는 코드
**사용 이유:** 클라이언트가 웹 서버에 요청을 했을 때 통신 결과를 웹 서버가 클라이언트에게 명확하게 전달하기 위해서

| 번호대 | 설명            |
| --- | ------------- |
| 1xx | Informational |
| 2xx | Succesful     |
| 3xx | Redirection   |
| 4xx | Client Error  |
| 5xx | Server Error  |

| 상태 코드 | 설명                                                                    | 예시                                    |
| ----- | --------------------------------------------------------------------- | ------------------------------------- |
| 200   | 요청이 성공적으로 처리 되었음                                                      | - 게시글 목록 조회                           |
| 201   | 요청이 성공적으로 처리 되었고, 그 결과로 새로운 리소스가 생성되었음                                | - 게시글 추가                              |
| 202   | 요청을 수신했지만, 그에 대한 처리를 할 수 없음<br>(다른 프로세스에서 처리 중이거나 서버가 요청을 처리하고 있는 경우) | - 게시글 추가<br>- 서버에서 처리에 시간이 필요한 경우     |
| 204   | 헤더는 유효하나, 요청에 대해 보내줄 수 있는 내용이 없음                                      | 게시글 업데이트                              |
|       |                                                                       |                                       |
| 400   | 잘못된 문법으로 인해 서버가 요청을 이해할 수 없음                                          | - 데이터가 서버로 전달이 안 된 경우                 |
| 401   | 인증되지 않는 접근                                                            | - 로그인 하지 않고 회원만 볼 수 있는 글을 읽으려 할 때     |
| 403   | 인가되지 않는 접근                                                            | - 관리자가 접근할 수 있는 페이지에 일반 유저가 접근하려고 할 때 |
| 404   | 요청받은 리소스를 찾을 수 없음                                                     | - 삭제된 게시글을 조회하려고 할 때                  |
| 429   | 지정된 시간 안에 너무 많은 요청을 보냈음                                               | - 사용자가 잘못된 비밀번호로 많은 요청을 했을 때          |
|       |                                                                       |                                       |
| 500   | 서버가 처리 방법을 알 수 없음                                                     | - 서버에서 처리할 수 없는 오류가 발생                |
| 501   | 요청 방법이 서버에서 지원되지 않아 처리할 수 없음                                          | - 지원하지 않는 HTTP메서드로 요청이 왔을 때           |
| 503   | 서버가 요청을 처리할 준비가 되지 않았음 (일시적 사용 불가능)                                   | - 서버에 많은 트래픽이 몰리거나 서버 유지보수 상황인 경우     |
