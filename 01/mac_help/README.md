# [과제 1] CLI 프로그램 제작
- Mac 단축키를 검색하는 CLI 프로그램입니다.
- 원하는 기능을 입력하면 해당하는 단축키를 출력합니다.
## How to Use
```
pip install -r requirements.txt
```
```
python mac_help.py [원하는 기능]
```

<summary><b>260514 - Levenshtein distance 기반 검색 추가</b></summary>

오타가 있거나 완벽하게 일치하지 않는 유사어여도 매칭해주기.

Difflib, rapidfuzz, 직접 구현 3가지 선택지를 비교함.

**Difflilib**

- LCS(최장공통부분 문자열) 알고리즘 기반


- 파이썬 기본 내장(Pip install 필요 없음)

- 데이터가 많아질 경우 상대적으로 느려짐

**Rapidfuzz**

- Levenshtein distance 기반

- C++으로 구현되어 있어 훨씬 빠른 속도

- pip를 통한 설치 필요

라이브러리를 사용하지 않고 직접 Levenshtein distance를 계산하는 함수를 구현해 사용하기로 결정하였는데, 그 이유는

1. LCS보단 Levenshtein distance가 적합 - query단어와 단축키 설명문 전체를 한번에 비교하지 않고, query단어와 단축키 설명문의 단어를 하나하나씩 비교하도록 로직을 생각했기 때문.

2. pip 설치 x - 가벼운 프로그램이라 따로 뭔가를 설치하고 dependency를 만들기 싫었음.

3. 소규모 데이터 - 데이터의 양이 많지 않아 속도가 덜 중요함.

4. 그냥 직접 짜고 싶었음 - 알고리즘 수업 때 짰던 기억이 나서 오랜만에 다시 짜보고 싶어짐

그래서 직접 Levenshtein distance 함수를 구현해 사용했으나 별 효과가 없었음.

ex) 캡"처"로 저장된 설명문을 캡”쳐”로 검색했으나 검색되지 않음.

그 이유는 단어 - 단어로 검색을 진행하다보니 전체 글자수가 많지 않고, 한글 특성상 캡쳐와 캡처는 매우 비슷한 단어지만 (1 - levenshtein거리 / 글자수)로 유사도를 계산하면 0.5가 나온다…

그래서 jamo라이브러리를 사용해서 각 단어를 자모단위로 분리한 후, 이 상태에서 다시 levenshtein distance를 구하는 방식으로 변경. 따로 라이브러리를 설치하기 싫었지만 결국 설치함..ㅜ

jamo 라이브러리를 통해 자모단위로 분리해서 다시 유사도를 구한 결과 기대하던대로 잘 동작하는 것을 확인함. LCS 알고리즘과 Levenshtein 거리를 모두 사용하면 보다 다양한 상황에서 원활하게 동작할 것 같은데, 내일 이 부분을 추가해봐야겠다.

<summary><b>260515 - 검색 로직 변경, 마무리</b></summary>

어제까지의 방식에는 쿼리가 2단어 이상일 경우가 문제가 있음을 확인했다.
쿼리를 단어단위로 쪼개고, 각 쿼리단어와 단축키 설명문의 단어 하나하나씩 levenshtein 거리를 구하고, 이들의 최댓값을 threshold와 비교해서 탐색 여부를 결정하였다.
대부분의 상황에서 잘 동작하는 것을 확인하여 과제를 제출한다. 이후에는 스크린샷 vs 캡처와 같은 유사어를 처리하는 작업을 추가할 수 있을 것 같다.
